# LG U+ Winning Tech Intelligence System (WTIS) v3.0 — 운영 지침서

> **목적**: 과제 제안서를 입력받아 최신 뉴스·특허·논문·시장 데이터를 체계적으로 수집하고,
> 레퍼런스 기반의 경쟁사 기술 분석 + 실행 타당성 보고서를 생성하는 인텔리전스 시스템.
>
> **v3.0 변경 사항**: Proposal 모드 신설, SKILL-0(제안서 파서) 추가, SKILL-4 레퍼런스 체계화,
> 에이전트 간 파일 핸드오프 프로토콜 명확화, 애자일 로드맵에 따른 확장 설계.

### v2.0 대비 변경

| 항목 | v2.0 | v3.0 |
|------|------|------|
| 진입점 | 자유 형식 쿼리 | 과제 제안서 지원 (Proposal 모드) |
| 스킬 수 | 5개 (SKILL 1~5) | 6개 (SKILL 0~5) |
| SKILL-0 | 없음 | 제안서 파서 — 구조화된 Analysis Brief 생성 |
| SKILL-4 | 뉴스 모니터링 중심 | 체계적 데이터 수집 + 레퍼런스 표준화 |
| 레퍼런스 | "출처 명시" 원칙만 | 표준 포맷 + 신뢰도 태그 [A/B/C/D] |
| 핸드오프 | 요약 텍스트 전달 | 파일 경로 전달 (Read로 로드) |
| 확장성 | 없음 | MCP 서버 연동 로드맵 (Phase 2~5) |

## 2. 모드별 파이프라인

### Proposal Mode (v3.0 핵심)
과제 제안서 → 분석 보고서 전체 파이프라인.

```
입력: 과제 제안서 (텍스트 or 파일)
  │
  ├─ SKILL-0: 파싱 → Analysis Brief
  │    출력: 키워드, 경쟁 대상, 검색 전략, 검증 필요 항목
  │
  ├─ SKILL-4: 데이터 수집
  │    입력: SKILL-0 Analysis Brief (파일 경로)
  │    출력: 기술현황, 경쟁사, 시장, 특허/논문, 규제 + References 테이블
  │
  ├─ SKILL-1: 선정검증
  │    입력: SKILL-0 Brief + SKILL-4 데이터 (파일 경로 2개)
  │    출력: SMART 검증, TRL 맵, 경쟁 비교, 3B 전략
  │
  ├─ SKILL-5: 교차검증
  │    입력: SKILL-1 결과 (파일 경로 1개)
  │    출력: 인용/수치/논리/편향 검증
  │
  └─ 최종 보고서 집계
```

### Quick / Standard / Deep
기존 v2.0과 동일. SKILL.md 참조.

## 3. 스킬별 역할과 경계

| 스킬 | 역할 | 하는 것 | 하지 않는 것 |
|------|------|---------|-------------|
| SKILL-0 | 제안서 파서 | 파싱, 키워드 추출, 검색 쿼리 생성 | 타당성 판단, WebSearch 실행 |
| SKILL-1 | 선정검증 | SMART, TRL, 경쟁 비교, 3B 전략 | 진행 관리, 신규 발굴 |
| SKILL-2 | 진행검증 | Gate Review, 리스크, Go/No-Go | 선정 판단, 3B 전략 변경 |
| SKILL-3 | 발굴 | 신규 과제 후보, 포트폴리오 분석 | 개별 과제 검증 |
| SKILL-4 | 데이터 수집 | 검색, 수집, 교차참조, 레퍼런스화 | 전략 제언 (시사점까지만) |
| SKILL-5 | 교차검증 | 인용/수치/논리/편향 검증 | 새로운 분석, 전략 변경 |

## 4. 레퍼런스 관리 체계 (v3.0 신규)

### 신뢰도 태그
```
[A] Official   — 기업 공시, 정부 통계, peer-reviewed 논문
[B] Reputable   — 주요 언론, 애널리스트 리포트
[C] Indicative  — 블로그, 소셜미디어, 미검증 보도자료
[D] Unverified  — 단일 소스, 루머, 익명 소스
```

### 레퍼런스 표준 포맷
```markdown
| # | 출처명 | URL | 발행일 | 접근일 | 신뢰도 |
|---|--------|-----|--------|--------|--------|
| [1] | Gartner Market Forecast | https://... | 2025-11-15 | 2026-02-24 | [B] |
```

### 인라인 인용
본문에서 `[1]`, `[2]` 형식으로 인용하고, References 테이블에서 상세 정보를 제공한다.

### 교차검증 기준
- 시장 규모 수치: **최소 2개** 독립 소스 필요
- 경쟁사 현황: **최소 1개** 공식 소스 (공시/IR) 필요
- 기술 성숙도: 논문 또는 공식 발표 기반

## 5. 파일 핸드오프 프로토콜

### 원칙
- 스킬 간 데이터 전달은 **파일 경로**로 한다 (텍스트 붙여넣기 X)
- 오케스트레이터는 이전 스킬의 `file_path`를 다음 스킬의 프롬프트에 포함
- 수신 스킬은 `Read` 도구로 해당 파일을 로드

### 파일 명명 규칙
```
research/sessions/{YYYY-MM-DD}_wtis-{mode}-{topic-slug}-{skill}.md
```
예시:
```
2026-02-24_wtis-proposal-ai-network-automation-skill0.md
2026-02-24_wtis-proposal-ai-network-automation-skill4.md
2026-02-24_wtis-proposal-ai-network-automation-skill1.md
2026-02-24_wtis-proposal-ai-network-automation-skill5.md
2026-02-24_wtis-proposal-ai-network-automation-final.md
```

### 필수 반환 필드
모든 스킬은 다음을 반환해야 한다:
```
status: pass | fail | uncertain
summary: (200자 이내 요약)
file_path: (저장한 파일의 절대 경로)
```

## 6. 분석 프레임워크 참조

### SMART 기준 (SKILL-1)
Specific, Measurable, Achievable, Relevant, Time-bound

### TRL 4사분면 (SKILL-1)
```
         High TRL (7~9)
              │
   [유지]     │     [베팅]
              │
──────────────┼──────────────
              │
   [탐색]     │     [Watch]
              │
         Low TRL (1~6)
   Low Disruption ←──→ High Disruption
```

### 3B 전략 (SKILL-1)
```
Build if: 차별화 중요도 > 8/10 AND 내부역량 보유 AND 시장여유 > 18개월
Buy if:   시장진입 긴급도 > 8/10 OR 기술격차 > 2년
Borrow if: 위 둘 다 해당 없을 때
```

### Gate Review (SKILL-2)
| Gate | 단계 | 통과 기준 |
|------|------|-----------|
| G0 | 아이디어 → 기획 | 고객 Pain Point 검증, 시장 규모 확인 |
| G1 | 기획 → PoC | 기술 실현가능성 ≥ 70% |
| G2 | PoC → 파일럿 | PoC KPI 달성률 ≥ 80% |
| G3 | 파일럿 → 상용화 | 파일럿 만족도 ≥ 4.0/5.0 |
| G4 | 상용화 → 확장 | MAU·매출 목표 ≥ 90% |

### 경쟁 동향 신호 (SKILL-2)
```
🟢 현행 유지: 경쟁사 유의미한 움직임 없음
🟡 속도 가속: 경쟁사 유사 제품 개발 착수 확인
🔴 피벗 검토: 경쟁사 출시 완료 또는 특허 선점
```

## 8. 운영 원칙

### 아키텍처
1. **오케스트레이터는 라우팅만** — 직접 분석 금지
2. **파일 기반 핸드오프** — 스킬 간 데이터는 파일 경로로 전달
3. **검증 독립성** — SKILL-5는 분석 컨텍스트 없이 실행
4. **비용 인식** — Quick으로 충분하면 Standard/Deep 불필요

### 데이터 품질
1. **정량 우선** — 모든 주장에 수치 + 출처
2. **교차검증** — 핵심 수치는 2개 이상 소스
3. **최신성** — 6개월 이내 데이터 우선
4. **신뢰도 태그** — [A/B/C/D]로 소스 품질 명시

### 출력 품질
1. **경영진 요약 1페이지** + 상세 분석 분리
2. **액션 지향** — Next Action 필수
3. **레퍼런스 체인** — 모든 주장 → 인용 번호 → References 테이블
4. **경쟁사 실명** — 추상 표현 금지

### 보안·윤리
- 공개 정보만 활용 (뉴스, 공시, 특허, 논문)
- AI 생성 분석임을 명시
- 최종 의사결정은 담당자가 검토
